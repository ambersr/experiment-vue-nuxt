## Experiment Vue/Nuxt

Dit project is een experiment met Vue en Nuxt en haalt data op uit een Directus API.  
Het doel is om een website te ontwikkelen die content dynamisch kan weergeven en te onderzoeken hoe de user, developer en content management experience is van deze techstack.

### User Experience
- UX van deze stack is over het algemeen goed op zowel desktop als telefoon en ipad.
- Toegankelijkheid is vrijwel vergelijkbaar als andere techstacks.

### Developer Experience
- Eenvoudige setup (uitgelegd in docs).
- Documentatie van Vue, Nuxt en Directus is uitgebreid en goed onderhouden.  
- Componenten zijn gemakkelijk te hergebruiken.
- Zonder TypeScript kun je de code minder goed checken/controleren.

Uitgewerkt in [Gist](https://gist.github.com/ambersr/6df731666f8af9e591f45b2571efcec1)

### Content Management Experience
- Content wordt opgehaald via Directus (heeft een overzichtelijke UI).
- Beheerders kunnen eenvoudig velden en content beheren zonder kennis van Vue/Nuxt.
- CMS is gemakkelijk te gebruiken en maakt contentbeheer onafhankelijk van developers.

Uitgewerkt in [Gist](https://gist.github.com/ambersr/63cefe45dd05a1468ce652b3402c1d7b)

## Conclusie gebruik techstack

Vue met Nuxt en Directus is geschikt voor websites die dynamisch en interactief moeten zijn. Developers hebben een overzichtelijke structuur en herbruikbare componenten. Contentbeheerders kunnen makkelijk content beheren via Directus zonder enige code kennis. Deze stack werkt goed voor projecten met interactieve functies en dynamische data maar is minder handig als je vooral een volledig statische en snelle website wilt.
